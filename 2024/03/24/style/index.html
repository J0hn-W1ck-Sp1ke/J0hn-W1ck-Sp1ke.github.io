比赛码风

比赛码风

命名规范

  • CamelCase (驼峰命名法)
    • 结构体、类的定义名首字母必须大写
    • 常量全大写
    • 变量名和函数名首字母小写
  • 名字太长可以截取前两、三个字母

代码块规范

  • 所有代码块的 \{\} ，必须按以下规范写 
while(a < b){//减少不必要空行，也避免大括号不匹配的问题
    if(a < b){
        for(int i=1; i<b; ++i){//注释可以直接写在这里，简洁明了

        }
    }
    else{

    }
}

int sum(int a, int b){//求和函数
    return a+b;
}

禁止使用

  • endl 超时必备
  • auto 旧版本C++不支持
  • 指针 平常都用不明白就别用了

必须使用

  • 关闭流同步
  • scanf 和 printf ，不允许使用另说

头文件、类型定义和函数规范

#include <iostream>
using namespace std;
typedef long long ll;//不要用define long long ll
const int N=100;//常量全大写定义

//不要用结构体内包含函数的形式，但是必须把数据和函数写在一起，并用注释简单注明该部分的作用，例如以下并查集

//并查集
int fa[N];//并查集结构体的数据成员
int find(int x){
    return x==fa[x] ? x : fa[x]=find(fa[x]);
}

//允许使用结构体包装数据，但是函数必须是全局的
int head[N], tot;
struct Edge{
    int v, nex;
}e[N<<1];
inline void add(int u, int v){//添加边
    e[++tot]=Edge{v, head[u]};//给结构体赋值的规范写法
}

int main(){

    //用注释和空格将步骤分开，简单写就行

    //这一步做什么

    //这一步做什么

}
