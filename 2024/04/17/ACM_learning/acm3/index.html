动态规划专题

发表于 2024-04-17 更新于 2024-06-12 分类于 算法模板 阅读次数：

动态规划

维度 -> 状态 -> 状态转移方程 -> 结果表和解决方案

思想与实现

核心思想与相关定义

通常基于问题的起始状态以及连续状态之间的循环公式或关系。问题的状态通常代表子解决方案，即部分解决方案或基于给定输入的子集的解决方案。并且状态是根据之前构建的状态一一构建的。

状态域 是定义了映射结果的状态集合

动态规划三要素：

  • 维度
    • 维度是状态的特征，受决策影响
    • 选择一个满足优化单调性/方案数的维度 作为 DP 结果，即状态域的输出，剩下的维度 作为状态域的输入描述具体的状态
    • 维度的信息是离散的下标（取值范围），状态是维度的组合所表达的语义信息
  • 状态
    • 基本子问题的解，即初始状态，是 DAG 的零入度点
    • 后续子问题的解，即中间状态，是 DAG 的中间节点
    • 整个问题的解，即最终状态，是 DAG 的零出度点
  • 状态转移方程
    • 动态规划本质上是一个DAG上的递推，按拓扑序求解保证无后效性
    • 根据不同的决策约束条件分类讨论状态转移，绘制状态转移图推导出状态转移方程
  • 问题的时间复杂度 = 状态域大小 计算每个状态所需时间

基本思路

  1. 列举所有维度，选择满足优化单调性/方案数的维度作为 DP 结果，剩下的组成状态
  2. 划分问题阶段，明确三个状态：提出基本子问题获取初始状态，提出后续子问题以获取中间状态，整个问题作为最终状态。确定解决思路（初始到中间到最终）
  3. 根据不同的决策约束条件，分类讨论状态转移方程
  4. 按拓扑序求解每一阶段的问题，即从初始状态经中间状态到达最终状态的最优路径

两种实现

  • 拓扑序递推：按拓扑序递推出结果
    • 我为人人（前向式 DP）
      • 符合思考习惯
      • 遍历所有动态规划状态，并从每个状态执行一些过渡，以向前到达其他状态
      • 每个过渡都会修改当前存储的一些未处理状态的结果。当考虑状态时，其结果已经完全确定
      • 状态关系是通过考虑状态的一个部分解并尝试将其延续到更大的状态而获得的
    • 人人为我（后向式 DP）：
      • 遍历所有状态，然后通过向后查看并使用先前状态的已知 DP 结果来计算每个状态的结果
      • 状态关系是通过检查状态的最佳解决方案并尝试将其分解为较小的状态来获得的
      • 在选取最优备选状态的值（人人的和或人人中的最值）时可以优化降低时间复杂度

  • 记忆化搜索

    • 记录已遍历的状态信息，只处理一次必要的状态
    • 只适用于后向式 DP

应用

优化问题

  • 根据目标函数选择最优可行解
  • 状态转移方程形式形如 ，即在几个候选方案中选择最好=最小的结果
  • 计算任何内容之前使用中性值（必不影响问题答案的结果）填充 DP 结果数组
    • 最小化问题中性值为正无穷大（大于数据范围），最大化问题中性值为负无穷大（0或小于数据范围）
    • 具有中性值结果的状态可以被认为是一种不可能的状态
  • DP 状态即 DP 子问题，因为它们的输入数据是整个问题输入的子集的某个问题
  • DP 方法依赖于最优子结构属性：给定整个问题的最优解，其部分解对于子问题也必须是最优的

组合问题

  • 计算做某事的方案数，或事件发生的概率或期望
  • 状态转移方程形式形如 ，求和意味着 F1、F2、…、Ft 情况中的不同方法总共构成了状态 (s) 的所有方法
  • 使用求和，中性值为 0 ，使用累乘，中性值为 1
    • 中性结果值意味着这种情况是不可能的
  • 关键
    • 对每种可能的方式进行计数
    • 并且不会对任何方式进行多次计数

递推问题

恢复优化问题（输出最优解决方案）

  • DP 解决方案路径从某个基本 DP 状态到某个最终状态，并且由代表所需最佳解决方案的所有部分解决方案的状态组成
  • 从结尾到开头重新计算 DP（只支持后向式 DP）
    1. 追踪路径的最终状态 (f)
    2. 寻找可以转移到 (f) 的中间状态 (p)
    3. 用 (p) 替代 (f) 重复第二步
    4. 找到初始状态 (s) ，结束路径追踪
  • 将反向链接与 DP 结果一起存储
  • 首选第二种方法，只有内存不够存储反向链接时才用第一种方法

优化

DP 问题的朴素解决方案可能会超出时间或空间限制，此时需要优化。对于 DP 问题，后向式 DP 当前状态的结果是通过对先前结果的几个公式进行求和/取最值操作计算得到的，一种优化方法是优化这些操作的时间复杂度。

合并等价状态（优化状态设计）

动态规划的原理是将任何具有大量等价状态的递归解合并，避免重复计算等价状态。如果递归解中没有等价状态，那么将其转化为动态规划是没有意义的。

如果问题答案以相同的方式依赖于两个状态，则这两个状态对于最终问题答案而言是等价的。如果 DP 的状态域包含许多组等价状态，则 DP 参数可能是多余的。所有等价状态都可以合并成单个状态。组合问题合并状态的结果必定是合并状态组上所有结果的总和，优化问题合并状态的结果必定是合并状态组上所有结果的最值。

修改不可能的状态 （优化状态设计）

参数即维度。

如果状态的结果始终为中性值，则该状态是不可能的，删除该状态不会影响问题的答案。不可能状态有以下来源：

  • 显式参数依赖：状态域 (A,B) 中 B=f(A) ，删除维度 B ，状态域化为 (A) ，需要 B 时使用 f(A) 即可
  • 隐式参数依赖：状态域 (A,B) 中 存在 f(A,B)=0 ，最好是将其中一个参数表示为明确依赖于其他参数，转化为显式参数依赖
  • 参数不等式：利用参数之间的不等式关系设置紧密的循环迭代边界，避免处理不可能的状态
  • 无需思考的方式
    • 丢弃不可能的状态，不处理它们：只需在遍历 DP 状态的循环内添加类似“if (res[i][j]==0) continue;”的内容即可。节省状态处理的时间
    • 使用带有记忆功能的递归搜索：只处理可能的状态
    • 将状态结果存储在映射中：不可能状态就不会消耗内存和时间。有序映射 map O(logN) ，无序列表 unordered_map O(1)

仅存储两层 DP 状态域的结果（滚动数组）

状态具有 (i, A) 的形式，其中 i 是 DP 分层的层索引，同时，状态转移方程仅取决于两个相邻层。

内存只包含当前层(i&1)和上一层((i-1)&1)，通过奇偶性交换，当前层完全处理之后，立即清除下一层((i+1)&1)的内容（设置为中性值）

如果 DP 转换关系使用不超过 k 个后续层，则包含 K 层，使用模 K 运算。

恢复路径而不存储所有 DP 结果的技巧：分治法

  1. 将层分为两半：设有一个包含多层的DP问题，将这些层划分为两半。例如，如果有N层，则选择中间层m（即第N/2层）
  2. 找到中间状态：执行标准的DP，计算到达每个状态的最优解。例如，从初始状态(i,A)出发，计算到达中间层m的所有状态的最优解
  3. 确定最终和中间状态：在DP完成后，确定最终层中的最优解（即最小结果）。设这个结果为R，并且它对应的中间状态为mA（在中间层m）。现在我们知道了最终状态和中间状态mA
  4. 递归处理每半部分：将层再次分为两半，分别处理这两半部分。对于左半部分，最终状态是中间状态mA；对于右半部分，初始状态是中间状态mA。对这两半部分再次执行DP，并找到新的中间状态
  5. 继续递归：重复上述步骤，直到处理到只有一层为止

预处理

  • 许多组合问题需要预先计算二项式系数。可以预先计算数组的前缀和，以便能够在 O(1) 时间内计算段中元素的总和。有时预先计算数字的前 k 次幂是有益的
  • 花费一些额外的时间来快速执行特定操作

旋转优化问题

对于一个最大化问题，有状态域 的 DP 解决方案，且同时要求 W 最小。若 R 的范围比 W 的范围小，则可以使用旋转优化后的状态域 ，此时通过坐标轴旋转将问题转化为最小化 W ，仅适用于 R 的范围小于 W的范围。

应用

数位DP

实现

  • 预处理最高 i 位为 j 的数出现的次数
    • 状态
      • 表示最高 i 位为 j 的数出现的次数
    • 状态转移方程
      • 第一个表示 i-1 位的贡献，第二个表示第 i 位的贡献
  • 预处理满 i 位每个数字出现的次数
    • 状态
      • 表示满 i 位每个数字出现的次数
    • 状态转移方程
  • 计数方法：
    • 将上界按位分开
    • 从高位到低位枚举，不贴着上界时低位随便取值，贴着上界时低位只能取到上界
    • 分两部分计算贡献(注意前导0)

思想

  • 数位是指把数字一位一位拆开，关注每一位上的数字
  • 计数问题中一个区间的答案可以拆成两个前缀和之差

应用

  • 统计满足一定条件的数的数量，条件转换后可用数位思想理解判断，输入给定数字区间限制统计范围
